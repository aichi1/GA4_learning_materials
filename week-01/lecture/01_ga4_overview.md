# GA4 とは何か — 概要とプロパティ設計

## この章の目標

- GA4（Google Analytics 4）が何者で、なぜ今これを学ぶのかを理解する
- ユニバーサルアナリティクス（UA）との違いを把握する
- GA4 プロパティの基本構成を理解する

---

## 1. GA4 とは

GA4（Google Analytics 4）は、Googleが提供するウェブ・アプリ向け行動計測ツールです。
2023年7月にユニバーサルアナリティクス（UA）が廃止され、現在の標準となっています。

### GA4 の特徴

| 特徴 | 説明 |
|------|------|
| イベントベース計測 | すべての計測単位が「イベント」。ページビューも一種のイベント |
| ウェブ＋アプリ統合 | 同一プロパティでウェブとモバイルアプリの計測が可能 |
| 機械学習の活用 | 予測指標（購入確率・離脱確率）を自動生成 |
| BigQuery 連携 | 無料で生データをエクスポート可能（今回は対象外） |
| プライバシー配慮 | Cookie 依存を減らし、モデリングで補完する設計 |

---

## 2. UA との主な違い

| 観点 | UA（旧） | GA4（現在） |
|------|---------|------------|
| 計測モデル | セッションベース | イベントベース |
| データ構造 | ヒット・セッション・ユーザー | イベントのストリーム |
| ページビュー | 独立した計測単位 | `page_view` イベントの一つ |
| 目標（コンバージョン） | 目標設定が複雑 | 任意のイベントをコンバージョンに指定可能 |
| レポート構造 | 固定レポート中心 | 探索レポートで柔軟にカスタマイズ可能 |
| データ保持 | 最大50ヶ月 | デフォルト2ヶ月（14ヶ月まで延長可） |

**重要な考え方の転換**: UAでは「ユーザーがページを訪れた」という発想でしたが、
GA4では「ユーザーが何らかのアクション（イベント）を起こした」という発想に変わります。

---

## 3. GA4 の階層構造

GA4 の管理単位を理解することは、正しい計測設計の第一歩です。

```
Googleアカウント
└── GA4 アカウント（組織・会社単位）
    └── プロパティ（サービス・サイト単位）
        └── データストリーム（計測するウェブ/アプリ単位）
```

### 各階層の役割

**アカウント**
- 組織または個人の最上位単位
- 複数のプロパティを管理する入れ物

**プロパティ**
- 計測データが蓄積される単位
- 1つのサービス（ECサイト）に対して1つが基本
- 測定ID（G-XXXXXXXXXX形式）がここに紐づく

**データストリーム**
- 実際のデータ送信元（ウェブサイトURL、モバイルアプリ）
- 1プロパティに複数追加できる
- ウェブストリームには「測定ID」が発行される

---

## 4. 測定ID（G-XXXXXXXXXX）とは

測定IDは `G-` で始まる識別子で、gtag.js を使ってデータを送信する際に必ず指定します。

```
例: G-ABCD1234EF
```

この測定IDを、後の handson で Next.js のコードに設定します。

---

## 5. プロパティ設計の考え方

1つのECサイトに対して、プロパティは基本的に1つで十分です。

### 環境ごとの分け方（推奨）

| 環境 | 推奨対応 |
|------|---------|
| 本番環境 | 本番プロパティ（G-XXXXXX-PROD） |
| 開発・ステージング | 別プロパティ or DebugView で確認 |

本番データに開発中のテストデータが混入すると分析精度が下がるため、
本番環境と開発環境は別プロパティにするか、DebugView を活用して分離します。

---

## 6. 自動収集されるイベント

gtag.js を設置するだけで、以下のイベントが自動的に収集されます。

| イベント名 | タイミング |
|-----------|-----------|
| `page_view` | ページが表示されたとき |
| `session_start` | セッション開始時 |
| `first_visit` | 初回訪問時 |
| `user_engagement` | ページが一定時間表示されたとき |
| `scroll` | ページを90%以上スクロールしたとき |
| `click` | 外部リンクをクリックしたとき |

Week 2 では、これらに加えてECサイト固有のカスタムイベントを実装します。

## まとめ

- GA4 はイベントベースの計測ツール。すべてのアクションがイベントとして記録される
- UA との大きな違いは「セッションベース → イベントベース」という計測モデルの転換
- プロパティ → データストリーム → 測定ID の階層を理解すれば、設定迷子にならない
- gtag.js を埋め込むだけで `page_view` など基本イベントは自動収集される

次の章では、セッション・ユーザー・エンゲージメントという概念を深掘りします。
