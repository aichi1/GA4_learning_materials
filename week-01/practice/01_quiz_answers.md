## 問題 1 の解答

**正解: B**

> イベントを基本単位として、すべてのユーザー行動を記録する

**解説**:

GA4 はイベントベースの計測モデルを採用しています。
ページビューも `page_view` というイベントの1つとして扱われます。
ユニバーサルアナリティクス（UA）はセッションベースでしたが、GA4 では
「ユーザーが何かアクションを起こした = イベントが発生した」という考え方が基本です。

選択肢 A は UA（旧 Google Analytics）の説明です。
選択肢 C・D は誤りです。GA4 はユーザーIDに依存せずに計測します。

## 問題 2 の解答

**正解: C**

> `usePathname` でパス変化を検知し、手動で `page_view` イベントを送信する

**解説**:

Next.js のような SPA では、ページ遷移時にブラウザが再読み込みされません。
そのため `gtag.js` を配置しただけでは、最初のページビューしか計測できません。

App Router では `usePathname`（`next/navigation` から）を使ってパス変化を監視し、
変化のたびに `gtag('config', gaId, { page_path: path })` を呼び出す必要があります。

選択肢 B の `useRouter().events` は Pages Router（旧 Router）の書き方で、
App Router では使用できません。

## 問題 3 の解答

**正解: D**

> セッション中に10秒以上滞在した、または2ページ以上閲覧した、またはコンバージョンが発生した

**解説**:

GA4 のエンゲージメントセッションの条件は以下の **いずれか1つ** を満たすことです。

| 条件 | 詳細 |
|------|------|
| 滞在時間 10秒以上 | ページがフォアグラウンドにあった時間 |
| 2ページ以上閲覧 | 複数ページへの遷移 |
| コンバージョン発生 | 設定したコンバージョンイベントが発生 |

選択肢 A（3ページ以上）と B（5秒以上）は閾値が誤りです。
選択肢 C（外部リンククリック）はエンゲージメントセッションの条件ではありません。

## 問題 4 の解答

**正解: B**

> Next.js の `next.config.js` で CSP ヘッダーに `*.google-analytics.com` を許可する

**解説**:

CSP（Content Security Policy）は、ブラウザがどのドメインへの通信を許可するかを制御する
セキュリティヘッダーです。CSP が設定されている場合、許可リストにないドメインへの
スクリプト読み込みやネットワークリクエストはブラウザによってブロックされます。

さくらVPS では CSP ヘッダーを手動で設定します（Vercel のように自動設定はありません）。
GA4 を計測するには以下のドメインを許可する必要があります。

```
script-src: https://www.googletagmanager.com https://www.google-analytics.com
connect-src: https://www.google-analytics.com https://analytics.google.com
```

GTM は今回使用しないため選択肢 A は誤りです。
Vercel は使用していないため選択肢 C は関係ありません。
GA4 管理画面にIPアドレス許可の機能はないため選択肢 D は誤りです。

## 問題 5 の解答

### 解答例

**1日目の訪問（8秒で離脱）:**

エンゲージメントセッションに **なりません**。

理由: エンゲージメントセッションの条件は「10秒以上滞在」「2ページ以上閲覧」「コンバージョン発生」のいずれかです。
1日目は8秒（10秒未満）、1ページのみ閲覧、コンバージョンなしのため、いずれの条件も満たしません。

**2日目の訪問（複数ページ遷移・購入）:**

エンゲージメントセッションに **なります**。

理由: 商品ページ → カートページ → 購入ページと3ページ閲覧（2ページ以上の条件を満たす）、
かつ購入イベントがコンバージョンとして設定されていれば、複数の条件を同時に満たします。

**バウンスについて（1日目）:**

1日目の訪問は「バウンス」として計測 **されます**。

GA4 のバウンスは「エンゲージメントセッションにならなかった訪問」として定義されています。
1日目はエンゲージメントセッションの条件を満たさないため、バウンスとなります。
