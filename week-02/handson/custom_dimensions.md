# カスタムディメンション設定手順

> **目的**
> `item_category` など GA4 の標準レポートに含まれないパラメータを
> 分析軸として使えるよう、GA4 管理画面に登録する。
>
> **所要時間**: 約15分
>
> **前提**: `gtag.ts` の実装が完了し、イベントが DebugView で確認できていること

---

## なぜカスタムディメンション登録が必要か

GA4 はイベントパラメータを**受け取る**が、**すべてを自動でレポートに表示するわけではない**。

GA4 の標準レポートに表示されるのは、Google が事前に定義した標準パラメータのみ。
`item_category` のような独自パラメータはイベントで送っていても、
管理画面でカスタムディメンションとして登録するまで分析に使えない。

```
コードで送っている → GA4 はデータを受け取っている → でも登録しないとレポートに出ない
```

---

## カスタムディメンションの種類

| スコープ | 説明 | 例 |
|---|---|---|
| イベントスコープ | 1つのイベントに紐づく | `item_category`（購入時のカテゴリ） |
| ユーザースコープ | ユーザーに紐づく（継続） | `membership_level`（会員ランク） |

今回は **イベントスコープ** の `item_category` を登録する。

> **上限**: 無料プランではイベントスコープのカスタムディメンションは **50個**まで。
> 不要になったものは削除して枠を空けることができる。

---

## 手順: item_category のカスタムディメンション登録

### ステップ 1: GA4 管理画面を開く

1. [GA4](https://analytics.google.com/) にアクセス
2. 左下の「管理」（歯車アイコン）をクリック
3. 「プロパティ」列の「**カスタム定義**」をクリック

### ステップ 2: カスタムディメンションを新規作成

1. 「カスタムディメンション」タブが選択されていることを確認
2. 右上の「**カスタムディメンションを作成**」ボタンをクリック

### ステップ 3: ディメンション情報を入力

以下の通りに入力する：

| 項目 | 入力値 | 説明 |
|---|---|---|
| ディメンション名 | `商品カテゴリ` | GA4 レポートで表示される名前。日本語でOK |
| スコープ | `イベント` | ドロップダウンから選択 |
| 説明（任意） | `商品のカテゴリ（tshirt, mugなど）` | メモ用 |
| イベントパラメータ | `item_category` | ★ コードで送っているパラメータ名と完全一致させる |

> **重要**: 「イベントパラメータ」の値はコードの文字列と**完全一致**でなければならない。
> `item_category` と入力したなら、`gtag.ts` でも `item_category` というキーを使うこと。

### ステップ 4: 保存

「保存」ボタンをクリック。

---

## 登録後の確認

### 即時確認はできない

カスタムディメンション登録後、レポートに反映されるまで **24〜48時間** かかる場合がある。
「登録したのにレポートに出ない」のは正常。翌日確認すること。

### DebugView での確認（即時）

DebugView では登録前でもパラメータの値が表示される。
`view_item` または `purchase` イベントをクリックして
`item_category` が送られているかを確認できる。

---

## 動作確認チェックリスト

```
[ ] GA4 管理画面 → カスタム定義 に item_category が表示されている
[ ] DebugView で view_item イベントの items[] に item_category の値がある
[ ] DebugView で purchase イベントの items[] に item_category の値がある
[ ] （翌日以降）探索レポートで「商品カテゴリ」ディメンションが選択できる
```

## よくある失敗と対処法

### 失敗1: 「イベントパラメータ」の入力値を間違えた

**症状**: レポートにディメンションは表示されるが値が空になっている

**原因**: コードでは `item_category` を送っているのに、カスタムディメンションの
「イベントパラメータ」欄に `itemCategory` などと入力してしまった（名前の不一致）

**対処**: カスタムディメンションを編集して正しいパラメータ名に修正する

---

### 失敗2: スコープを「ユーザー」にしてしまった

**症状**: イベントに紐づいた分析ができない

**原因**: `item_category` はイベントごとに変わるのにユーザースコープで登録してしまった

**対処**: 削除して「イベントスコープ」で再登録する

---

### 失敗3: コードでパラメータを送っていない

**症状**: DebugView を確認しても `item_category` が表示されない

**原因**: `gtag.ts` の `GaItem` に `item_category` を含めていない、
または呼び出し時に渡していない

**対処**: `sendViewItem` / `sendAddToCart` / `sendPurchase` の呼び出しを確認し、
`item_category` を渡していることを確認する

```typescript
// 正しい例
sendViewItem({
  item_id: product.id,
  item_name: product.name,
  price: product.price,
  quantity: 1,
  item_category: product.category,  // ← これが必要
})

// NG 例（item_category が抜けている）
sendViewItem({
  item_id: product.id,
  item_name: product.name,
  price: product.price,
  quantity: 1,
})
```
