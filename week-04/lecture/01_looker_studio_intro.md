# Looker Studio 入門 — なぜ Looker Studio を使うのか

## 概要

Looker Studio（旧称 Google Data Studio）は、Google が提供する**無料のデータ可視化ツール**です。
GA4 と直接接続でき、分析結果をダッシュボードとして可視化・共有できます。

---

## 1. なぜ Looker Studio を使うのか

### 1-1. GA4 UI の限界

GA4 の管理画面は分析用途には優れていますが、次のような場面で限界があります。

| 課題 | 内容 |
|------|------|
| レポートの固定 | GA4 のレポートは毎回同じ操作が必要 |
| 共有の難しさ | GA4 管理画面へのアクセス権がないと見られない |
| カスタマイズ制限 | KPI の組み合わせや表示順を自由に変えにくい |
| プレゼン向きでない | クライアントに見せる資料としては使いにくい |

### 1-2. Looker Studio が解決すること

- **ダッシュボードを URL で共有できる** — クライアントや上司にリンクを渡すだけで見てもらえる
- **自動更新** — GA4 データはリアルタイムに近い形で反映される
- **自由なレイアウト** — KPI に合わせてグラフ・数値・表を自由に配置できる
- **複数データソース対応** — GA4 + スプレッドシートなど組み合わせも可能

## 2. 料金

| プラン | 料金 | 制限 |
|--------|------|------|
| 無料版 | **無料** | ダッシュボード数・コネクタ数に上限なし（Googleコネクタは無料） |
| Looker Studio Pro | 有料（エンタープライズ向け） | チーム管理機能・SLA等 |

**個人学習・副業ポートフォリオ用途は無料版で十分です。**

---

## 3. GA4 との連携の仕組み

Looker Studio は GA4 のデータを**コネクタ（データソース）**を通じて取得します。

```
GA4 プロパティ
    ↓ （Google Analytics コネクタ）
Looker Studio データソース
    ↓
ダッシュボード（グラフ・数値・表）
```

### ポイント: GA4 の「プロパティ」単位で接続する

GA4 にはビュー（UA時代の概念）がありません。
接続する単位は**プロパティ**（例: `GA4 - EC_project_sdd_v7`）です。

---

## 4. Looker Studio の主要コンポーネント

| コンポーネント | 説明 | 用途 |
|--------------|------|------|
| **スコアカード** | 単一の数値を大きく表示 | セッション数・購入数など KPI の強調 |
| **時系列グラフ** | 日付軸で推移を表示 | トレンドの把握 |
| **棒グラフ** | カテゴリ比較 | チャネル別・デバイス別など |
| **表** | 行×列で詳細を表示 | ページ別・流入元別など |
| **フィルタ** | 絞り込み条件を設定 | 期間・セグメント切り替え |
| **日付範囲コントロール** | 表示期間をインタラクティブに変更 | — |

```
└─────────────────────────────────────────────────┘
```

## 6. Looker Studio の基本操作フロー

1. [https://lookerstudio.google.com/](https://lookerstudio.google.com/) にアクセス
2. Google アカウントでログイン（GA4 と同じアカウント推奨）
3. 「空のレポート」から作成
4. データソースとして GA4 プロパティを追加
5. グラフ・数値・テキストを配置してダッシュボードを構成

---

## 7. ポートフォリオとしての価値

Looker Studio ダッシュボードを副業提案に活用することで、次のことをクライアントに証明できます。

- GA4 でのデータ収集設計と実装ができる
- データを可視化してビジネス判断に使える形にできる
- ダッシュボードを URL で共有でき、レポーティングの負担を削減できる

**「GA4を入れた」で終わらず、「分析して見せる」までできることが差別化ポイントになります。**
