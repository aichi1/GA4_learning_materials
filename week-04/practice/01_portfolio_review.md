# ポートフォリオ自己評価チェックリスト

## 使い方

4週間のカリキュラムを完走したら、このチェックリストで自己評価を行ってください。
全項目に ✅ をつけることが最終ゴールです。
未チェックの項目は復習・やり直しの優先順位として活用してください。

---

## Week 1: GA4 基礎・gtag.js 実装

### 実装完了

- [ ] GA4 プロパティを作成し、測定 ID（`G-XXXXXXXXXX`）を取得した
- [ ] `GoogleAnalytics.tsx` コンポーネントを作成し `layout.tsx` に組み込んだ
- [ ] `lib/gtag.ts` に `pageview` 関数を実装した
- [ ] ローカル開発環境で GA4 DebugView にイベントが届いていることを確認した
- [ ] さくらVPS 本番環境でも GA4 が動作していることを確認した

### 理解確認

- [ ] GA4 の「プロパティ」「データストリーム」「測定 ID」の違いを説明できる
- [ ] `gtag.js` と `Google Tag Manager (GTM)` の違いを説明できる
- [ ] `layout.tsx` に直接埋め込む方式を選んだ理由を説明できる

## Week 2: eコマースイベント計測設計・実装

### 理解確認

- [ ] `gtag('event', ...)` の呼び出し方法と必須パラメータを説明できる

## Week 3: GA4 分析スキル

### 分析操作

- [ ] GA4 管理画面でファネル探索レポートを作成した
- [ ] `view_item → add_to_cart → purchase` のファネルで各ステップの通過率を確認した
- [ ] チャネルグループ別のセッション数を「集客 → トラフィック獲得」で確認した
- [ ] セグメント（例: 購入者 vs 非購入者）を作成して比較分析を行った

### 洞察の引き出し

- [ ] ファネルのどのステップで離脱が多いかを特定した
- [ ] 流入チャネルの中で CVR が高いチャネルを特定した
- [ ] 分析結果をもとに改善施策を1つ以上提案できる

## Week 4: Looker Studio ダッシュボード

### 接続・設定

### ダッシュボード完成

- [ ] チャネル別セッションの棒グラフを配置した
- [ ] 購入ファネル（view_item → add_to_cart → purchase）を追加した

### 共有・ポートフォリオ化
