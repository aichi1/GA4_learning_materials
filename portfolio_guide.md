# ポートフォリオ活用ガイド — 副業提案への組み込み方

## はじめに

このガイドでは、本カリキュラムで作成した GA4 実装・Looker Studio ダッシュボードを
副業の提案や転職・フリーランス営業で活用する方法を説明します。

---

## 1. 副業提案文への書き方

### 基本方針

「GA4を入れた」という事実だけでなく、**何を計測して何を判断できるようにしたか**を伝えることが重要です。
クライアントは「データを収集した」よりも「意思決定に使えるレポートを提供してくれる」を求めています。

### 提案文テンプレート（GA4実装経験の書き方）

---

**【テンプレートA: 実績として記載する場合】**

```
■ GA4実装・アクセス解析（ECサイト）

Next.js + さくらVPS 構成の EC サイトに Google Analytics 4 を自前実装しました。
Vercel 等のマネージドサービスを使わない環境でも gtag.js を直接組み込む形で
カスタムイベント（商品閲覧・カート追加・購入完了）まで計測できる状態にしました。

▼ 実績
・ eコマースイベント設計〜実装（view_item / add_to_cart / purchase）
・ Stripe webhook と連携した purchase イベント発火
・ Looker Studio によるKPIダッシュボード作成（セッション・CVR・チャネル別売上）

▼ 成果物
・ Looker Studio ダッシュボード（URL 共有可能）
・ イベント設計書（Markdown）
```

---

**【テンプレートB: 提案として売り込む場合】**

```
■ 提案内容: GA4導入〜ダッシュボード整備

現在、サイトのアクセス状況・購入率・流入チャネルがリアルタイムで把握できる
環境を構築するご支援が可能です。

▼ 対応範囲
1. GA4プロパティ設定・タグ実装（Next.js / その他フレームワーク）
2. 購入ファネルの計測設計と実装（目標設定・eコマースイベント）
3. Looker Studio でのKPIダッシュボード作成
4. URL共有可能なレポート提供（更新自動）

▼ 実績サンプル
・ ダッシュボードサンプル: [Looker Studio URL]（閲覧専用）
```

---

### 提案文で避けるべき表現

| 避けるべき表現 | 改善後 |
|--------------|--------|
| 「GA4を設置できます」 | 「eコマースイベントまで含めた計測設計〜実装が可能です」 |
| 「アクセス解析が得意です」 | 「セッション・CVR・チャネル別の KPI ダッシュボードを提供できます」 |
| 「Looker Studioを使えます」 | 「更新自動・URL共有可能なレポートを月次コストゼロで提供できます」 |

## 2. ダッシュボードの URL 共有設定

### 手順

Looker Studio ダッシュボードを外部に共有するには「閲覧者」権限での共有設定が必要です。

**重要: 「閲覧者」設定にしないとURLを渡せません**

1. Looker Studio でダッシュボードを開く（編集モード）
2. 右上の「**共有**」ボタンをクリック
3. 「**リンクを取得**」タブを選択
4. アクセスを「**リンクを知っている全員**」に変更
5. 権限を「**閲覧者**」に設定（編集者にしないこと）
6. 「**リンクをコピー**」をクリック
7. コピーしたURLを提案文・ポートフォリオサイトに貼り付け

### 共有設定のパターン

| 共有相手 | 推奨設定 | 理由 |
|---------|---------|------|
| 面接・提案先クライアント | リンクを知っている全員（閲覧者） | アカウント不要でアクセスできる |
| 長期クライアント | 特定のGoogleアカウントに共有 | セキュリティを確保できる |
| ポートフォリオサイト掲載 | リンクを知っている全員（閲覧者） | 訪問者が即座に確認できる |

> **注意**: 閲覧者は編集・データのエクスポートができません。クライアントが閲覧するだけなら閲覧者で十分です。

## 3. スクリーンショット戦略

URL を渡せない場合、またはポートフォリオ資料（PDF・スライド）に載せる場合はスクリーンショットを使います。

### 撮影前の準備

1. **個人情報・実売上の非表示**
   - 実際のサイトのデータを使う場合: 売上金額・注文数などの実数値を隠す
   - 方法: Looker Studio の「フィルタ」でデータを絞るか、GA4 デモアカウントを使用
   - ダッシュボード上に「サンプルデータ（Google Merchandise Store）」と注記を入れる

2. **表示期間の設定**
   - 「過去 28 日間」など適切な期間を選択してからスクリーンショットを撮る
   - データが少ない場合は期間を広げる（過去 90 日など）

3. **ブラウザの拡大率を調整**
   - 80〜90% に縮小するとダッシュボード全体が一画面に収まりやすい

### 撮影と加工

1. **全体スクリーンショット**: ダッシュボード全体をキャプチャ
   - Mac: `Cmd + Shift + 3`（全画面）/ `Cmd + Shift + 4`（範囲選択）
   - Windows: `Win + Shift + S`（範囲選択）

2. **セクション別スクリーンショット**: KPI サマリー・ファネルなど見せたい部分を切り抜き

3. **注釈を追加（推奨）**
   - Figma・PowerPoint・Keynote などで赤枠・吹き出しを追加
   - 「週次 CVR が X% → 施策実施後 Y% に改善」など成果を添える

### スクリーンショットの使用場面別ガイド

| 使用場面 | 推奨枚数 | 重点的に見せるもの |
|---------|---------|-----------------|
| クラウドソーシング（ランサーズ・クラウドワークス）プロフィール | 1〜2枚 | ダッシュボード全体 |
| 提案書（PDF） | 2〜3枚 | 全体 + ファネルのクローズアップ |
| ポートフォリオサイト | 3〜5枚 | 全体・KPI・ファネル・チャネル別 |
| 面接資料 | 3〜4枚 | 実装の難しかった点を説明できる箇所 |

## 4. 副業プラットフォーム別の活用方法

### クラウドソーシング（ランサーズ・クラウドワークス）

- プロフィールの「得意分野」に「GA4実装・Looker Studio」を追記
- ポートフォリオ作品として Looker Studio の URL またはスクリーンショットを登録
- 説明文に「Next.js への自前実装経験（GTM不使用）」を入れると差別化になる

### SNS（X / LinkedIn）

- ダッシュボードのスクリーンショットを投稿し「#GA4 #Looker Studio #副業」などのタグをつける
- 「実装で詰まった箇所」や「CSP設定のtips」など技術的な知見を発信すると信頼度が上がる

### ポートフォリオサイト / GitHub

- `README.md`（本リポジトリのルート）に実績として記載
- Looker Studio ダッシュボードの URL をリンクとして貼る
- イベント設計書（Week 2 成果物）も資料として公開すると「設計もできる」証明になる

## 5. よくある質問

**Q: 本番サイトのデータを見せても良いですか？**

A: 実売上・個人情報が含まれる場合は避けてください。
次のいずれかを使ってください。
- GA4 デモアカウント（Google Merchandise Store）のデータ
- 自分のテストデータ（架空の購入データ）
- 実数値を隠して「前週比 +XX%」など相対値のみ表示

**Q: ダッシュボードが英語になっています**

A: Looker Studio の言語設定を確認してください。
設定 → 全般 → 言語 → 日本語に変更。指標・ディメンション名も日本語化されます。

**Q: クライアントに「GA4の導入費用」を聞かれたら？**

A: GA4・Looker Studio 自体は無料です。工数（設計・実装・レポート整備）で費用が発生します。
Week 2 のイベント設計書を見積もりの根拠として活用できます。
