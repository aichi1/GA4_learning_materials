# Week 3 Handson: 完了条件

## 概要

Week 3 の handson では、GA4 の管理画面を実際に操作して
ファネル分析・チャネル分析・セグメント比較を行い、
分析レポート 1 枚を完成させます。

---

## 前提条件

- Week 2 の実装が完了していること
  - `view_item` / `add_to_cart` / `purchase` イベントが GA4 に届いている
- Week 2 実装から **最低 24〜48 時間**が経過していること
  - GA4 のデータは翌日以降にレポートへ反映されます
  - DebugView ではリアルタイム確認可能ですが、探索レポートとは別物です

---

| 順序 | ファイル | 内容 | 目安時間 |

## 完了条件

以下をすべて満たしたら Week 3 handson 完了です。

### 必須条件

- [ ] **ファネルデータ探索** を GA4 で作成し、3ステップの離脱率を確認した
  - `view_item → add_to_cart → purchase` のファネルが表示されている
- [ ] **チャネル別データ** でトップ3チャネルのセッション数・CV率を確認した
- [ ] **新規 vs リピーター** のCV率を数値で比較した
- [ ] **分析レポート（`04_analysis_template.md`）** を埋めて 1 枚完成させた
  - セクション 1〜5 がすべて記入されている
  - 「観察 → 仮説 → 施策」が少なくとも 1 セット書かれている

### 推奨条件（余裕があれば）

- [ ] ファネルデータ探索でデバイス別（スマホ vs PC）の通過率を比較した
- [ ] ソース / メディア別のデータを確認した（Direct が多い場合に UTM 未設定を疑えるか）
- [ ] セッションセグメントとユーザーセグメントの結果の違いを確認した

## 注意事項

### データがほとんどない場合

Week 2 の実装直後でデータが少ない場合は、
`practice/01_data_interpretation.md` の仮データを使って分析レポートを作成できます。
「仮データ使用」と明記してレポートを完成させてください。

### 探索タブが見つからない場合

GA4 管理画面の左サイドバーにある「探索」アイコン（コンパス形）をクリックします。
「レポート」（棒グラフ形）とは別の場所にあります。
サイドバーをスクロールしても見つからない場合は、権限が「閲覧者」以上であることを確認してください。

### 数値が 0 や (not set) ばかりの場合

- イベント実装から時間が経っていない → 翌日以降に再確認
- `view_item` / `add_to_cart` のスペルが GA4 の標準イベント名と一致しているか確認
- DebugView（管理 → DebugView）でイベントが届いているか確認
