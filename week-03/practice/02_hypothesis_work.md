# 施策仮説立案シート

このワークでは「観察 → 仮説 → 施策」の思考フレームを使って、
データから施策のアイデアを導き出す練習をします。

---

## 思考フレームの説明

```
観察（Observation）
   ↓ 「なぜそうなっているのか？」
仮説（Hypothesis）
   ↓ 「だとしたらどんな施策が有効か？」
施策（Action）
   ↓
検証（Verify）← 施策を実行して GA4 でデータを再確認
```

- **観察**: データが示している「事実」を記述する。解釈は含めない
- **仮説**: 観察の「原因・理由」を推測する。仮説は複数立ててOK
- **施策**: 仮説が正しいと仮定した場合に取れる具体的なアクション
- **検証方法**: 施策の効果をどの指標で確認するか

> **重要**: 仮説は「間違えてもいい」ものです。
> 大切なのは「データを見て考える習慣」を身につけること。
> 施策を実行してデータが変化すれば仮説が正しかった、
> 変化しなければ別の仮説を立てるというサイクルが分析の本質です。

---

## ワーク 1: ファネルデータから仮説を立てる

以下のデータを使って思考フレームを埋めてください。
（practice/01_data_interpretation.md の仮データ 1 を参照）

```
ファネルデータ:
  view_item:   12,400 ユーザー
  add_to_cart:  2,480 ユーザー（通過率 20%）
  purchase:       744 ユーザー（通過率 30%）
```

---

### シート 1-A（記入例）

| セクション | 内容 |
|---|---|
| **観察** | 商品詳細を見たユーザーの 80% がカートに追加せず離脱している |
| **仮説 1** | 商品価格が競合より高く、購入を躊躇させているのではないか |
| **仮説 2** | 商品説明文が不足しており、購入判断に必要な情報が足りないのではないか |
| **仮説 3** | スマートフォンでの商品ページが見づらく、カートボタンが押しにくいのではないか |
| **採用仮説** | 仮説 3（スマートフォンのUX問題） → まずデバイス別ファネルで確認可能 |
| **施策** | ① デバイス別ファネルをGA4で確認 ② スマホ画面でカートボタンのサイズ・位置を改善 |
| **検証方法** | 改善後 2 週間のスマホユーザーのカート追加率と比較 |

## まとめ: 思考フレームの活用場面

| 場面 | 使い方 |
|---|---|
| 週次レビュー | 毎週 GA4 を見て気になった数値に対してシートを1枚記入 |
| 施策提案 | クライアントへの改善提案書にこのフレームを使う |
| A/B テスト設計 | 「仮説 → 施策」をそのままテスト設計に転用 |
| ポートフォリオ | 「データを見て考えられる人」であることを証明する実績として使える |

このシートを埋めたものが Week 3 の分析レポート（handson/04_analysis_template.md）の
骨格になります。
